# Changelog

## 0.8.0.dev0 — working Phase 8 checkpoint

- Reframed the package as a measured track-based drivetrain design framework.
- Added `drivetrain-study` while preserving the `cvt-study` alias.
- Added doctor, result index, report regeneration, cache inspection/clear, and
  conservative prototype-event migration commands.
- Added deterministic parallel study execution, content-addressed summary
  caching, atomic result publication, per-scenario checkpoints, resume/restart,
  progress, and ETA.
- Added physical energy attribution, uncertainty screening attribution, decision
  synthesis, hierarchical reports, decision traces, provenance JSON/SVG, and
  machine-readable appendices.
- Added model registry/protocol boundaries for future drivetrain and tire models.
- Expanded user, input, output, developer, methods, release, and worked-example
  documentation.

This is a working checkpoint reconstructed from the reviewed Phase 6 base after
the original transient Phase 8 download failed. See
`WORKING_PHASE_8_STATUS.md` for validation scope.
